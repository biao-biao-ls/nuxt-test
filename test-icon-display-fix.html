图标显示修复对比测试

修复前（问题版本）

  • 图标颜色为灰色 (#999)
  • SVG尺寸20x20可能显示不全
  • 内联fill属性覆盖CSS

修复后（正确版本）

  • 图标颜色为白色
  • SVG尺寸调整为24x24
  • 使用!important强制覆盖
  • 移除内联fill属性

问题分析

问题1：内联fill属性覆盖CSS SVG中的 fill="#999" 内联样式优先级高于CSS的 fill: white 问题2：SVG尺寸不合适 原来：width="20" height="20" 在28px容器中可能显示不完整 修复：width="24" height="24" 更好地填充容器 问题3：CSS选择器优先级不够 原来：.open-leftbar-icon svg { fill: white; } 修复：.open-leftbar-icon svg { fill: white !important; } .open-leftbar-icon svg path { fill: white !important; }

修复方案

1. 移除SVG内联fill属性 - 从 <path fill="#999"> 改为 <path> - 让CSS样式能够生效 2. 调整SVG尺寸 - 从 width="20" height="20" 改为 width="24" height="24" - 在28px容器中有更好的显示效果 3. 增强CSS选择器 - 添加 !important 确保样式优先级 - 同时设置svg和path的fill属性 4. 清理不必要的属性 - 移除 t、version、p-id 等不必要的SVG属性 - 保持代码简洁
